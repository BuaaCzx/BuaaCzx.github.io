### sizeof

-  ```
    int i= 0;
    printf(“sizeof(i++) = %d\n”, sizeof(i++));
    现在i等于几了？
    i的值还是0，什么原因？
    ```

    在C语言中，`sizeof` 是一个编算符（unary operator），用于计算数据类型或变量的大小（以字节为单位）。`sizeof` 在编译时就确定了其结果，并不实际执行其操作数。因此，在表达式 `sizeof(i++)` 中，尽管 `i++` 是一个后缀自增操作，但 `sizeof` 并不会导致 `i` 的值发生变化。 `sizeof` 操作并没有真正去执行 `i++`，它只是简单地返回了 `int` 类型的大小，通常情况下是4字节（在32位系统上）或者8字节（在某些64位系统上，但这取决于具体的编译器和平台设置）。

### 字节 Byte

#### 1个byte一定等于8个bit吗？

<limits.h>定义了一个宏CHAR_BIT，C语言中规定一个byte等于CHAR_BIT个bit。

C语言的标准规定CHAR_BIT>=8，1个Byte包括至少（而不是正好）8个bit。

byte（字节）是C语言最重要的基础存储单元，每一个字节都有一个编号，也就是地址。内存就是一系列byte顺序排列在一起组成的存储结构。

#### 我们机器里的内存有多少个这样的byte呢？我们说8/16/32/64位机到底是什么意思？

32位机，表示32位bit来表示byte的编号。可以编号的byte个数一共可以有232个，$2^{32}=2^{2}*2^{10}*2^{10}*2^{10}$
4G内存，其中1K=1024Byte，1M=1024K, 1G=1024M

#### 关于地址

地址是字节在内存中的编号，只有字节才有地址。我们一般谈论地址的时候指的都是字节的逻辑地址

### 对象

#### 对象是什么

对象由一个或多个连续字节序列组成

数据存储区域的内容可以表示值

#### 对象表示

对象各个bit组成的二进制串就是这个对象的对象表示（Object Representation），对象的值和对象表示通过对象类型可以进行映射

![image-20241128151622497](./../images/2024-11-28-C%E8%AF%AD%E8%A8%80%E7%B3%BB%E7%BB%9F%E7%BA%A7%E7%BC%96%E7%A8%8B/image-20241128151622497.png)

#### 对象表示vs 对象的值

如果有两个对象，对象类型一样：

1. 对象表示一样、则对象的值一定一样

2. 如果对象的值一样，对象表示则不一定一样

这是因为组成对象表示的bit中，包括Value Bits和Padding Bits。一个对象可能有若干个无意义的Padding Bits。Value Bits和Padding Bits的构成也是由对象类型决定的

#### 对象类型T及其大小和对齐要求的含义（sizeof，alignof）

C语言中的任何一个对象类型都可以形式化定义成一个T

sizeof(T)和alignof(T)

- sizeof(T): 规定了一个T类型的对象其大小是多少

  注：如果T是不完全对象类型，就没有size（后续介绍）

- alignof(T): 规定了一个T类型的对象其缺省对齐要求是多少
  注：这里用的是缺省对齐要求，意味着对象的对齐要求可变

####size, sizeof

对于任何一个对象来说，假设它包含的字节个数是n（n≥1）

n是这个对象的size，这个对象包含的比特数量就是n×CHAR_BIT个

假设这个对象的类型是T，则sizeof(T)就规定了T类型的对象字节个数（Size）

#### 对象地址，对齐要求，alignof

对象地址（Address）：对象所占用的连续字节中lowest字节的编号

对齐要求（Alignment）是要求对象的地址能被对齐要求整除

假设这个对象的类型是T，则alignof(T)就规定了T类型对象的缺省对齐要求

C23之前是_Alignof，C23开始提升为alignof

#### 标准有符号整数类型的相关概念

对于任何一个标准有符号整数类型T的对象来说，所有的bits分成3个部分：1个sign bit，若干个value bits，若干个padding bits。

![image-20241128153124855](./../images/2024-11-28-C%E8%AF%AD%E8%A8%80%E7%B3%BB%E7%BB%9F%E7%BA%A7%E7%BC%96%E7%A8%8B/image-20241128153124855.png)

示意图，sign bit、value bits和padding bits的摆放根据编译器要求

假设sign bit加上value bits的个数是N

N是这个对象类型T的宽度（Width），而N-1则是这个类型T的精度（Precision）

T类型对象的对象值范围：$-(2^{N-1})$到$2^{N-1}-1$

#### signed char类型的宽度

<img src="./../images/2024-11-28-C%E8%AF%AD%E8%A8%80%E7%B3%BB%E7%BB%9F%E7%BA%A7%E7%BC%96%E7%A8%8B/image-20241128160329992.png" alt="image-20241128160329992" style="zoom:50%;" />

例如：signed char类型对象，对象值范围：$-(2^{{SCHAR\_WIDTH}-1})$到$2^{{SCHAR\_WIDTH}-1}-1$

因为CHAR_BIT最小值是8，signed char类型的对象，其对象值范围至少是-128 ~ 127

sign char类型不允许有padding bits，且因为sign char类型的宽度一定是CHAR_BIT，因此sign char类型对象的size一定是1，sizeof(signed char)一定等于1

#### int类型的宽度

<img src="./../images/2024-11-28-C%E8%AF%AD%E8%A8%80%E7%B3%BB%E7%BB%9F%E7%BA%A7%E7%BC%96%E7%A8%8B/image-20241128160628093.png" alt="image-20241128160628093" style="zoom:50%;" />

例如: int类型对象, 对象值范围: $-(2^{INT\_WIDTH-1})$到$2^{({INT\_WIDTH}-1)}-1$ 

因为INT_WIDTH最小值是16, int类型的对象, 其对象值范围至少是-32768 ~ 32767

因为宽度宏至少是16, 因此sizeof(int)在不同平台上结果可能不一样 

除了signed char, 其他4种标准有符号整数类型也可以没有padding bits 

这里我们假设所有标准有符号整数类型都没有padding bits

#### typedef

`typedef`是一个能够为类型设置别名的关键字。

```
typedef T alias
```

假设有一个平台A (ImplementationA)，short int类型是2个字节，int类型是4个字节

假设另一个平台B (ImplementationB)，int类型是2个字节，long int类型是4个字节

可以通过`typedef`来提高程序的可移植性

使用`INT32`而不是`int/long int`来提高跨平台的移植性

代码示例：

```c
#ifdef ImplementationA
    typedef short int INT16;
    typedef int INT32;
#elifdef ImplementationB
    typedef int INT16;
    typedef long int INT32;
#endif
```

#### _BitInt(N)

C23提供了一个新类型，指定有符号整数类型的宽度为N。

一个_BitInt(N)类型的对象，对象值范围：$-(2^{N-1})$到$2^{(N-1)}-1$

_BitInt(3)和\_BitInt(4)不是同一个类型。

_BitInt(N)类型占用的字节数需要用sizeof来获得。

_BitInt(N)类型就有可能有padding bits了。

例如：\_BitInt(17): sizeof(_BitInt(17)) × CHAR_BIT - 17，计算padding bits
